# Mitigation vs Root Cause Fix

> <- Back to [[Incident Response]]

During an incident, the priority is to stop the bleeding (mitigation) first, then fix the root cause properly later. Mitigation actions like rollbacks, feature flag toggles, or traffic shifting restore service quickly. Root cause fixes come from post-incident analysis.
