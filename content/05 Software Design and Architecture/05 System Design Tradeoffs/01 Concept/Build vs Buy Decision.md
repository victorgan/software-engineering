# Build vs Buy Decision

> <- Back to [[When to Choose What]]

Build when it is a core competency, unique differentiator, or no good existing solution exists. Buy or use OSS for commodity problems, well-served by existing tools, or when the team lacks domain expertise. Speed to market often favors buying.
