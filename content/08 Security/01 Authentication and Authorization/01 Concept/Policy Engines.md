# Policy Engines

> <- Back to [[Authorization Models]]

External services that evaluate authorization policies: OPA (Open Policy Agent) uses Rego language, Cedar (by AWS/Verified Permissions) uses a purpose-built language, and Casbin supports multiple models. Policy engines decouple authorization logic from application code.
