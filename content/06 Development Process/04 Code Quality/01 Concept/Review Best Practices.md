# Review Best Practices

> <- Back to [[Code Review]]

Small PRs, clear descriptions, constructive feedback. Keep pull requests small (under 400 lines when possible). Write clear descriptions explaining the why. Give constructive, specific feedback. Approve with comments when blocking is unnecessary.
