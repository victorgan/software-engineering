# Graceful Degradation

> <- Back to [[Reliability Patterns]]

Providing reduced functionality instead of total failure when components are unavailable or overloaded. For example, serving cached content when the database is down, or disabling non-essential features under load. Users experience reduced quality rather than complete outage.
