# Ship Show Ask

> <- Back to [[Git Workflows]]

Categorize changes by review needs. Ship: merge directly (trivial changes). Show: merge then notify team (straightforward changes). Ask: open PR for review (complex or risky changes). Reduces review bottlenecks by right-sizing process to risk.
